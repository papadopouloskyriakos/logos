# Instructions for expert annotators (Linear B administrative roles)

You are one of **two independent** specialists assigning an **administrative role** to each Linear B
word-form in your packet. Work from established Linear B scholarship.

## Rules
1. **Annotate independently.** Do **not** communicate with the other annotator.
2. **Do not force a label** — use `UNKNOWN`, `AMBIGUOUS`, `MULTILABEL`, or set the **abstain flag** when the
   role is not determinable.
3. **Use established Linear B scholarship** (Ventris–Chadwick *Documents*²; Aura Jorro *Diccionario
   Micénico*; DĀMOS; standard editions).
4. **Cite the basis** for every `GOLD_A` or `GOLD_B` judgment (`source_citation` column).
5. **Preserve uncertainty** — record `alternative_labels`, `uncertainty_type`, and `dispute_flag`.
6. **Do NOT consult Linear A** (this is a Linear B task).
7. **Return the completed response template WITHOUT changing item IDs or their order.**

## Coarse roles (choose one per item)
`HUMAN_OR_INSTITUTION` · `PLACE` · `COMMODITY_OR_COUNTED_CATEGORY` · `MEASURE_OR_QUANTITY` ·
`OPERATOR_OR_RELATION` · `QUALIFIER` · `DOCUMENT_STRUCTURE` · `UNKNOWN` · `AMBIGUOUS` · `MULTILABEL` · `EXCLUDED`.
Optionally add a **fine role** and an **entry relation** (DESTINATION/ORIGIN/LOCATION/SOURCE/…) where supportable.

## Gold tiers
`GOLD_A` secure (uncontested) · `GOLD_B` probable (cite one alternative) · `GOLD_C` disputed/sensitivity-only
· `GOLD_X` excluded/uninterpretable.

## Deliverable
Fill `annotator_<A|B>_response_template.csv` and return it. Your packet is
`annotator_<A|B>_packet.csv` (reading + document context + neighbours + numerals/logograms + series/site/
support/hand + source citation). The two packets contain the **same items in different order**.
