## What it does

- **predict.py** — commits a §A comparison-hypothesis: canonical body → `plan_hash = sha256(body,
  sort_keys)` with `created_at` OUT (idempotent), thesis IN as `thesis_sha`, confidence IN. Fail-
  CLOSED gate: confidence ∈ (0,1]; **model sources (`embedding_nn`/`llm_proposed`) REFUSED above
  0.75** (not a silent clamp — invariant 5). Idempotent `INSERT … ON DUPLICATE KEY UPDATE` (first-
  freeze provenance wins). The `prediction` JSON column freezes the held-out evaluation set
  (`heldout_forms`, `candidate_lexicon`, `n_eff`, `u_floor`, `virgin_sign_support`,
  `lit_index_hit`, `null_recalls`) BEFORE the verdict.
- **verdict.py** — the **SOLE writer of `verdicts`** (invariant 2/3; grep-verified; no
  `anthropic`/`openai`/`llm_call` in the file). Mechanical grading: **deflated S_lex** (lexstat)
  primary per F.1 (S_morph gold-standard-if-corpus-cooperates; Linear A too short/formulaic);
  the **L_fake canary** (calibrated to the candidate) is the headline falsifier; the
  `corrected_margin_bar` (Cornish-Fisher + Bonferroni/DSR); DSR secondary via
  `logos_stats.deflated_sharpe` over the instrumented `N_eff`. The **§E gate = ALL of**:
  registered-before-test, lfake_null_present, `dsr≥0.95`, `k≤u_floor` (MDL), beats_lfake_margin,
  generalizes_to_virgin, `not(llm_proposed AND lit_index_hit)`. result: match/partial/deviation;
  gate_verdict: **GRADUATE / NULL_PUBLISHED / REJECT**; `code_sha` provenance in every row.
- **family_scores.py** — aggregates verdicts → `family_scores` + `graduation_state` (MIN_VERDICTS
  freeze).
- **logos_db.py** — mirrors `agora_lib` (load_env + db); confirmed live round-trip (MariaDB
  11.6.2; 7 tables; all comparison-layer columns present).

## Significance + gaps

The logos discipline machinery is **LIVE**: hash-keyed falsifiable hypotheses → mechanical held-out
verdicts → §E graduation gates that refuse unless every clause holds, with the LLM never grading
itself. This is the load-bearing core of logos's contribution.

Gaps (future work): a comparison **scanner** (the thing that generates hypotheses to feed
`predict`) is not yet built; the §E gate's `lit_index_hit`/`virgin_support` clauses are wired but
their data feeds (the literature index + `L_known`/`L_virgin` partition + LLM-ablation — the
remainder of #15) come next; `S_morph` is deferred per F.1.
