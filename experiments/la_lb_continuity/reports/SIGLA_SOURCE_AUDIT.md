# SIGLA_SOURCE_AUDIT — §IV

_All counts generated by `src/sigla_reconcile/reconcile.py` + `scripts/sigla_decode.py` (main repo);
figures in `data/RECONCILE_SUMMARY.json`. Regenerate: `python3 …/reconcile.py`._

## Source & access
- **Source:** SigLA — *The signs of Linear A: a palaeographical database* (E. Salgarella & S. Castellan),
  <https://sigla.phis.me/>. Snapshot `corpus/bronze/sigla_browse_2026/` (main repo), fetched **2026-07-03**.
- **Access method:** NOT an API. The whole corpus ships to every visitor **embedded in `database.js`**
  as **two OCaml `Marshal` blobs** (`var signs`, `var data`), OCaml-escaped (`\ddd` decimal) inside JS
  string literals. Decoded locally with `scripts/sigla_decode.py` (no browser, no OCaml runtime):
  JS+OCaml unescape → OCaml Marshal reader (blocks/ints/strings/doubles + object sharing) →
  `Map.Make`/list/option walkers. Both blobs parse to 100 % of their bytes.
- **License:** **CC BY-NC-SA 4.0** (site footer). → raw snapshot **and** decoded JSON are treated as
  licensed vendor-derived data, **gitignored** (`corpus/bronze/*`); this pass commits code + checksums
  + counts only.
- **Version:** site changelog states "all tablets from GORILA are now present" (2021-08-24) + explicit
  post-GORILA additions (2026-05-21, 2026-06-26). No semantic version string; pinned by input sha256
  (see `data/manifests/reconcile_inputs.sha256`).
- **Deterministic acquisition:** `database.js` (sha pinned) → `sigla_decode.py` → `sigla_documents.json`
  (968 KB) + `sigla_signs.json` (41 KB). Reproducible byte-for-byte from the pinned snapshot.

## Completeness (decoded)
- **802 documents** across **22 sites** (Haghia Triada 372, Khania 220, Phaistos 64, Zakros 44,
  Knossos 37, Mallia 20, Arkhanes 10, Kea 7, …).
- **376 signs** — **77 AB-class** (shared LA/LB homomorphs, `AB##`) + **299 A-only** (`A###`).
- **5,144 glyphs total, 4,756 sign-identified** (⇒ 388 dividers/unidentified). Each glyph carries its
  sign-id **and a bounding box** `[x,y,w,h]` in the drawing; **783/802** docs have a `source_url`,
  **634/802** carry physical dimensions.

## Identifiers & representation
- **Inscription id:** designation string (`"HT 24a"`), also `sigla_path` (`document/HT 24a`).
- **Sign id:** `class ∈ {AB, A}` + GORILA number → `AB81`, `A308`. Validated vs SigLA's own rendered
  `doc_ARKH7.html` (→ `A308`) and vs silver (HT 24a AB-signs 9/9).
- **Allographs:** the sign catalogue keys on canonical sign-id; per-glyph drawings/bboxes carry the
  palaeographic variation (not yet clustered into named allograph classes here — deferred to §V).
- **Damaged / uncertain:** represented **per glyph** — `sign=null` (unidentified) or `is_divider=true`
  (word divider). SigLA does not expose a separate "restored/uncertain" flag in the decoded payload;
  uncertainty therefore reads as *unidentified glyph*, not as a graded confidence.
- **Alternative readings:** each decoded glyph resolves to exactly **one** sign-id (no candidate list
  in the payload) — SigLA's palaeographic judgement is already collapsed to a single identification.
- **Provenance to GORILA:** sign numbering is GORILA; documents map to GORILA via designation; the
  post-GORILA additions are enumerated in the changelog (recorded in the snapshot PROVENANCE).
- **Duplicate/fragment handling:** SigLA **splits by face/impression** (e.g. roundel `HTWA1019Α`,
  `HTWA1019Γ` with Greek-letter faces) — a *finer* granularity than silver on roundels/nodules; joins
  are kept as separate members (silver uses `HT123+124`). This is the dominant crosswalk-granularity
  axis (see SIGLA_CORPUS_DELTA).

## Fitness for this pass
SigLA is a **sound, deterministic source of sign identity + palaeographic provenance**, suitable for
the §V A↔B sign-equivalence layer and as a coverage extension. It is **not** a drop-in replacement for
silver: the corpora overlap on ~575 inscriptions but each holds a large exclusive set, and they differ
in segmentation and face/join convention (SIGLA_CORPUS_DELTA). Silver remains the LA transcription
baseline (it is what the frozen slot classifier consumes); SigLA augments sign identity and palaeography.
